# Research: Autotools Scaffold

**Feature**: 001-autotools-scaffold  
**Date**: 2026-03-22

## 1. C++20 Detection via AX_CXX_COMPILE_STDCXX

- **Decision**: Use `AX_CXX_COMPILE_STDCXX(20, noext, mandatory)` from
  the GNU Autoconf Archive.
- **Rationale**: Portable, compiler-agnostic detection. Automatically
  injects `-std=c++20` (not `-std=gnu++20`). The `mandatory` parameter
  causes configure to abort with a clear error if C++20 is unavailable.
- **Alternatives considered**:
  - Direct `AC_LINK_IFELSE` with hardcoded `-std=c++20` — brittle
    across compilers.
  - Compiler-specific feature checks — non-portable.
- **Key details**:
  - The `.m4` file must be vendored into `m4/` (not auto-downloaded).
  - `configure.ac` must include `AC_CONFIG_MACRO_DIRS([m4])` before
    the macro invocation.

## 2. Libtool Library Naming with Hyphens

- **Decision**: Use `lib_LTLIBRARIES = libazure-storage-client.la` in
  `src/Makefile.am`. Automake converts hyphens to underscores for
  variable prefixes, so per-target variables use
  `libazure_storage_client_la_*`.
- **Rationale**: Automake's "uniform naming" scheme handles this
  automatically. The output `.so`/`.dll` preserves the hyphenated name
  (`libazure-storage-client.so.0.0.0`), matching the constitution and
  repository name.
- **Alternatives considered**:
  - Underscored library name (`libazure_storage_client`) — diverges
    from constitution and repo name.
  - Manual Makefile rules — bypasses Automake automation.
- **Key details**:
  - `libazure_storage_client_la_SOURCES = azure-storage-client.cpp`
  - `libazure_storage_client_la_LDFLAGS = -version-info 0:0:0`
  - `libazure_storage_client_la_LIBADD = $(LIBCURL_LIBS) $(OPENSSL_LIBS)`

## 3. pkg-config `.pc.in` Template

- **Decision**: Ship `azure-storage-client.pc.in` at repository root,
  generated by `AC_CONFIG_FILES([azure-storage-client.pc])`.
- **Rationale**: Standard pkg-config pattern. `configure` substitutes
  `@prefix@`, `@libdir@`, `@includedir@`, `@PACKAGE_VERSION@` at
  build time—no manual path maintenance.
- **Alternatives considered**:
  - Hardcoded `.pc` file — breaks when prefix changes.
  - CMake config files — incompatible with Autotools-only project.
- **Key details**:
  - `Requires: libcurl openssl` propagates transitive deps.
  - `Libs: -L${libdir} -lazure-storage-client`
  - `Cflags: -I${includedir}`
  - Installed to `$(libdir)/pkgconfig/` via Makefile.am
    `pkgconfig_DATA`.

## 4. Dependency Detection (libcurl, OpenSSL)

- **Decision**: Use `PKG_CHECK_MODULES` for both:
  ```
  PKG_CHECK_MODULES([LIBCURL], [libcurl])
  PKG_CHECK_MODULES([OPENSSL], [openssl])
  ```
- **Rationale**: Modern, standard approach. Every major distribution
  ships `.pc` files for libcurl and OpenSSL. Provides `*_CFLAGS` and
  `*_LIBS` variables automatically. Supports user overrides via
  `./configure LIBCURL_CFLAGS=... LIBCURL_LIBS=...`.
- **Alternatives considered**:
  - `LIBCURL_CHECK_CONFIG` — curl-specific; less portable.
  - `AC_CHECK_LIB` — requires manual flag assembly; no version info.
- **Key details**:
  - Detected variables: `LIBCURL_CFLAGS`, `LIBCURL_LIBS`,
    `OPENSSL_CFLAGS`, `OPENSSL_LIBS`.
  - Propagated to `src/Makefile.am` via
    `libazure_storage_client_la_CXXFLAGS` and
    `libazure_storage_client_la_LIBADD`.
  - No minimum version pinned for the scaffold; can be added later.
